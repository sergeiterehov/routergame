# RouterGame - симулятор сетей

Попробуй здесь: [sergeiterehov.github.io/routergame](https://sergeiterehov.github.io/routergame/)

На этом проекте я закрепляю знания по компьютерным сетям, попутно реализуя протоколы и паттерны в виде симулятора.

В качестве прикладной задачи - визуальный конструктор сети. Возможно, это будет некой игровой формой...

![Demo](demo.png)

Каждое устройство из сети работает через WebWorker, а обмен данными между ними идет через основной поток и отображается в журнале обмена.

Порты устройств и драйвера знают о наличии линка (флаг RUNNING). Наличие линка определяет регулярное прохождения маячкового сигнала - аналог Normal Link Pulse.

Есть простая файловая система. Она позволяет хранить настройки. Файловая система в реальном времени синхронизируется с основным потоком, поэтому можно управлять файлами запущенного устройства из UI.

Неуправляемое устройство - L2 Switch.

Управляемые устройства с операционной системой.
- Key-Value файловая система
- Ethernet
- Bridge
- VLAN
- ARP
- IP routing
- IPIP tunnel
- IPIP over UDP, NAT traversal
- ICMP
- UDP
- TCP
- NAT stateful
- Firewall
- Socket: raw, udp, tcp
- DHCP
- DNS
- HTTP

Все в OS работает на уровне ядра. Доступны некоторые упрощенные аналоги утилит:
- `cat`, `ls`, `rm`, `touch` - для работы с файлами
- `init` загрузка и оболочка терминала
- `pkg` - установка пакетов программ
- `netd` - демон-конфигуратор сети
- `net` - CLI для управления конфигурацией
- `iface` аналог ifconfig
- `route`
- `br` - создание мостов и управление vlan фильтром
- `arp` - просмотр и запрос mac адресов
- `ping`
- `trace` - аналог traceroute
- `socket` - просмотр активных сокетов
- `nc` - аналог netcat
- `dig`
- `bind8` - DNS сервер
- `dhclient` - запрос к DHCP серверу
- `dhcp_server` - DHCP сервер
- `fw` - управление firewall
- `url` - аналог curl
- `nginy` - NGINX-подобный веб сервер
- `tun` - создание туннелей

Управлять сетевой конфигурацией можно 2 способами:
1. императивно - команды типа `iface`, `route`, `br` меняют параметры сетевого стека прямо в оперативной памяти ОС;
2. декларативно - демон `netd` при старте применяет сохраненное состояние, а изменения делаются командами.
